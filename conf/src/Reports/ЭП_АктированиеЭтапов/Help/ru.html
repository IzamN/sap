Назначение отчета

Отчет предназначен для анализа проектов компании, включая обычные проекты и проекты сопровождения.


Варианты отчета

  • Актирование этапов


Условия формирования отчета


Отчет выводит этапы проектов, которые удовлетворяют одному из следующих условий:

    1. Этап проекта был помещен в архив в течение периода формирования отчета.
    2. Дата окончания этапа попадает в период формирования отчета.
    3. Этап не был добавлен в архив и его дата окончания меньше даты окончания периода формирования отчета.


Для проектов сопровождения отчет также выводит нераспределенные трудозатраты по техническим проектам, хотя бы один этап которых был выведен в отчет, если:

    1. Технический проект был добавлен в архив, и его дата окончания попадает в период формирования отчета.
    2. Технический проект не был добавлен в архив.

Цветовая индикация поля "Выручка"

    •     Оранжевый - этап указан в акте реализации, акт выпущен (проведен).
    •     Зеленый - этап указан в акте реализации, акт оплачен (установлена дата оплаты).
    •     Желтый - этап указан в акте реализации, акт получен (установлена дата получения).
    •     Белый - для этапа не указан акт или он не проведен.

Техническая реализация


В отчет попадают все этапы проектов не сопровождения, удовлетворяющие условиям выше и не являющиеся помеченными на удаление (РазделыПроекта, Проект):

  • Поля "Проект сопровождение", "Проект" заполняются из экземпляров справочника "Проекты"
  • Поля "Этап", "Дата начала этапа", "Дата окончания этапа (по договору)", "Плановые трудозатраты", "Плановая выручка", "Плановый субподряд" заполняются из экземпляров справочника "РазделыПроекта"
  • Поле "Плановая валовая прибыль" вычисляется как: "Плановая выручка" минус "Плановый субподряд"
  • Значение для поля "Фактические трудозатраты" берется из регистра "УчетВремени" по измерению "Проект" (при расчете не участвуют сотрудники с договором ГПХ)
  • Поле "Дата окончания этапа (прогнозная)" берет значение из регистра сведений "ЭП_ОкончаниеЭтапов" на основе:
    • Этапа проекта
    • Текущей даты
    (значение вносится в регистр из формы элемента "РазделыПроекта")
  • Поле "Фактическая выручка" заполняется на основе регистра "ЭП_РеализацияУслуг" по этапу
  • Поле "Фактический субподряд" заполняется из регистра "ЭП_ПоступлениеУслуг" по этапу
  • Поле "Фактическая валовая прибыль" вычисляется как: "Фактическая выручка" минус "Фактический субподряд"


В отчет попадают все этапы проектов сопровождения, удовлетворяющие условиям выше и не являющиеся помеченными на удаление (РазделыПроекта, Проект):

  • Поля заполняются аналогично проектам не сопровождения из тех же источников
  • Особенности для проектов сопровождения:
    • Поле "Фактические трудозатраты" берется из регистра "УчетВремени" по измерению "ТехническийПроект"
    • Поле "Дата окончания этапа (прогнозная)" берется из акта (не является реквизитом технического проекта)
    • Поле "Фактический субподряд" заполняется из регистра "ЭП_ПоступлениеПроектыСопровождения" по измерению "ТехническийПроект"


В отчет попадают проекты сопровождения без этапа, удовлетворяющие условиям выше:

  • В таких строках заполняются только поля:
    • "Проект"
    • "Проект сопровождение"
    • "Фактические трудозатраты" (рассчитывается как сумма значений из регистров):
      • "УчетВремени"
      • "ЭП_ПоступлениеПроектыСопровождения"
      для технических проектов, подчиненных проекту, у которых не указан акт реализации
